Bitcoin Security Engineer Learning Roadmap – From Fundamentals to Custody Systems (12 Months)

📅 12-Month Bitcoin Security & Custody Roadmap

📆 Months 1–2 — Bitcoin & Wallet Fundamentals

🎯 Goal: Understand Bitcoin’s core data model and how wallets and policies are constructed.
Learn:
  • UTXOs, inputs/outputs, and transaction structure
  • ECDSA & Schnorr signatures
  • BIP-32 HD wallets and BIP-39 mnemonics
  • PSBT (BIP-174) and how multi-party signing works
  • Output descriptors and Miniscript policies
  • Time-locks and hash-locks
📚 Resources:
  • Mastering Bitcoin – Chapters 4–7
  • Bitcoin Dev Guide
  • BIP-32
  • BIP-174
  • Miniscript site
🧪 Projects:
  • Manually build and broadcast transactions on regtest.
  • Construct and spend from a 2-of-3 multisig.
  • Create a descriptor with a timelock and test spending conditions.

📆 Months 3–4 — Rust & Bitcoin Libraries

🎯 Goal: Get comfortable building Bitcoin logic in Rust.
Learn:
  • Rust basics: ownership, borrowing, traits, enums, error handling
  • rust-bitcoin, rust-miniscript, and bdk
  • Programmatic PSBT creation and signing
  • RPC communication with Bitcoin Core
📚 Resources:
  • The Rust Book
  • rust-bitcoin
  • rust-miniscript
  • BDK Docs
🧪 Projects:
  • CLI tool to generate descriptors and build PSBTs.
  • Sign transactions programmatically in Rust.
  • Build a simple wallet backend on regtest.

📆 Months 5–6 — Secure Key Management & Threat Modelling

🎯 Goal: Learn how private keys are generated, stored, and protected.
Learn:
  • Entropy and randomness best practices
  • BIP-39 mnemonic → seed → BIP-32 key derivation
  • Key encryption (AES-GCM, libsodium)
  • Threat modelling (STRIDE) and secure design
  • HSM fundamentals and PKCS#11
  • Intro to MPC (GG18, FROST)
📚 Resources:
  • NCC Group Key Management Cheat Sheet
  • AWS CloudHSM
  • PKCS#11 Spec
  • ZenGo MPC Intro
  • FROST paper
🧪 Projects:
  • Build a secure key generator that encrypts keys at rest.
  • Implement a simple N-of-M signing service.
  • Enforce policies like time delays or dual approval.

📆 Months 7–8 — HSMs, MPC & Hardware Integration

🎯 Goal: Integrate with real-world secure signing environments.
Learn:
  • HSM operation and PKCS#11 APIs
  • Cloud key management (AWS KMS, GCP KMS)
  • Secure enclaves (TPM, Nitro Enclave)
  • MPC signing flows and threshold signatures
📚 Resources:
  • AWS CloudHSM Tutorials
  • GCP KMS Docs
  • Fireblocks MPC Deep Dive
🧪 Projects:
  • Build a signing service using an HSM simulator.
  • Simulate MPC by splitting a key across two services that must cooperate to sign.

📆 Months 9–10 — Custody Architecture & Policy Enforcement

🎯 Goal: Build institutional-grade security systems.
Learn:
  • Custody workflows (withdrawal delays, whitelists, approvals)
  • Policy engines and signing orchestration
  • Tamper-evident audit logging
  • Compliance standards: SOC2, FIPS 140-2/3, MiCA
📚 Resources:
  • BitGo Custody Overview
  • Fireblocks Developer Docs
  • Anchorage Resources
  • FIPS 140-2
  • SOC2
🧪 Projects:
  • Build a custody backend with multisig, time-delays, whitelists, and approvals.
  • Add tamper-evident logs of all signing attempts.

📆 Months 11–12 — Advanced Topics & Future-Proofing

🎯 Goal: Finish strong with advanced topics and portfolio projects.
Learn:
  • Post-quantum cryptography basics (Kyber, Dilithium)
  • Hybrid signature approaches (ECDSA + PQC)
  • Custody migration strategies
  • Advanced threat modelling and disaster recovery
📚 Resources:
  • NIST Post-Quantum Project
  • pqcrypto crate
🧪 Final Portfolio Projects:
  • 🏦 Toy Custody System – policies, approval workflows, secure signing
  • 🔐 Secure Key Vault – encrypted BIP-32 key storage with PKCS#11
  • 🪙 Policy-Based Signing Daemon – PSBT signing with conditions
  • 📜 Audit Tool – descriptor verification and backup validation

🧰 Portfolio & Career Path

By month 12, you’ll have:

  • ✅ Deep knowledge of Bitcoin security primitives
  • ✅ Rust experience with descriptors, PSBT, and Miniscript
  • ✅ Hands-on key management and policy-enforced signing
  • ✅ A working custody backend project
  • ✅ Knowledge of compliance and threat modelling
  • ✅ Awareness of post-quantum challenges
